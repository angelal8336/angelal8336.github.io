Player game

Context

As a final project in APCSA, we were asked to create a working game involving the use of most concepts such as methods, condionals, loops, and object orentated code. The game we created was called players in which the users battle aganist each other until one player is left. The main functions of the code include:

  1. Players can attack themselves and the damage is between 25 to 50
  2. When players chooses the sleep option, they heal a health between 1 to 25
  3. If players do not choose, they miss a turn

Challenges

  1. Getting the attack method to work in the player class was the by far the hardest thing to code besides getting started. It was a method that was used to attack another random player including itself when called in the runner file. Unlike other methods in the player class like sleep(), this one included a object player parameter called attack. Figuring out how to use this to subtract a random players health was difficult. In fact even understanding that we needed an object parameter was hard. However, with some help especially Mr Mueller, i figured out how to access the health variable of the object using a dot operator and by using `Math.Random()` was able to generate a random health to subtract from the original health.
  2. The second hardest thing I encountered was completing the extracredit that involved the use of inhertance. I had to understand the concept of super and childclass which I haven't really fully understood. In fact, in the beginning, I had no clue even how to start so I decide to google the basic of inhertance to view some examples. I even visited projectstem unit 9 coding assignment to get an understand on where to begin. When I understood it, I was able to code the basic of it which allowed me to complete my project. During the method coding, I had some issues with adding a buff method in the swordman childclass. It was sucessfully created but it run into an error in the runner file because it couldnt access it. to solve this, I asked the teacher for help and eventually figured out that I can simply create a "useless" method in the player class with the same name so that I can overwrite it. This allowed it to work, and I was able to finally finish my EC

Takeaways

  1. Asking someone for help can help you understand a code better as having someone who understands the code explain it, allows you to process in different perspective
  2. Use google or revisit pervious lessons if needed. If your stuck with a small part you can always use google to view examples to understand where to begin and how something work. You can also revisit assigments you completed to refresh and review the concept. As long your not using it to help you code the whole project.
