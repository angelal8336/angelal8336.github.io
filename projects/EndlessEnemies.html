Endless Enemies

Context

  • For the last SEP class I am taking in HighSchool, I created one last final freedom project using any tool and language I wanted even if we learned java the whole year. Out of all the things I could of made, I choose to make a game yet again but this time using goDot. I partnered with Shelly to create a 3D survival game in which the only goal was to gain as much points as possible by killing the mobs that spawn. Your only objective was to swing the sword and defeat the mobs one by one until you die. There is a scoreboard on top to keep track of your points and a deathscreen when you finally lost. Below is the challenges I encountered, takeaways and lastly what you been waiting for, a few links to our game

Challenges

  1. Collobrating on this project on goDot and github was the worst thing that happened during this project. It was so bad that I started hating working with others due to it. Since GoDot was not designed for a two person team, it was extremely hard to figure out how exactly to push and pull each other's work. In fact, even with the help of github, whenever we pulled or pushed, the other person did not get the code we coded. In other words, we were basically coding for nothing or for ourselves becasue we couldnt merge our code, or even get one another's. As you can imagine this made it SUPER HARD because we split the assignment into task. I was focused on creating the map and when I completed it, Shelly could not get my code or my scenes no matter how hard she tried. This meant that my map was esstiently lost and did not make it into the final product. For some of my other codes and her's we had to manually give each other the code through discord so that we can copy and paste it onto our sides. This came with a lot of problems as you can imagine but eventualy we found a point where we had the MVP and some beyond MVP done.
  2. Learning goDot was very difficult espically when you were trying to do 3D. There were so many componments on goDot engine and documentation to read that it took us so long to even understand a single portion of the tool. In fact, I probably send hours just looking through videos and documentation just to get a tiny grasp on how to use goDot. I was learning why coding my game at the same time. There was just so many things related to goDot and other tools that i needed to use like asset stores to compplete my game, that I need to learn. Every step of the way, I was confused on what to do. For example, I didn't understand how I could resize my scenes or make it so that the player can hit the mobs. It was so confusing and hard to learn the tool that I almost gave up but I didn't. In the end from this eperience, I started to understand how hard learning something on your own is. It isn't easy but VERY VERY difficult since you have to spend your own time to learn.
  3. Time management was another big issue in this project. We had the whole year to do it and to pace ourselves but I defintely did not do well at it. I had my schedule done on when to do what, but I lacked motiviation to complete it. Even if I did do it on the assigned day, it would take way longer than I expected. This made me constantly change my schedule,, pushing each day back until I couldnt do so anymore. Mangaing your time alone was already diffuicult but now I have to do so with a partner which made it even harder. We needed to consider each other's task so we can create a plan that works for the both of us. However, the main issue was my procasintation as I couldnt bring myself to actually work on it when I wanted or needed to. I did eventually work on it when I knew that I really had to and there was no other choice. This allowed to atleast finish the project but not to extent of what I planned. However, I'm still grateful that I eventually got myself to work.

Takeaways

  1. Don't procastination! If you leave work till the end to do then you will definitely be stressing and pulling all nighters just to get it done. If you have time and you know it is due soon, work on it a week before it is due. This will allow you time to adjust and change your plan is things go wrong
  2. Learning on your own is very tough so take it slowly. I think what I did was try and learn goDot all at once which did not work out. It made me super confused every step i took. If I had taken more time to learn it slowly, I think it would have been more easy to learn the tool.
  3. Always check the tool before using. We didn't really do research on goDot before we choose to use it, so we didn't know that it was bad which collobratin. If we knew, we would of choosen another tool that supported working as a team or just did our project by ourselves.
